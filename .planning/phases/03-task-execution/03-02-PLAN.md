Command body -- implement these four modes:

MODE 1: /task (no arguments) -- Show pending task queue per D-08 (review and approve)

1. Read data/tasks/active.jsonl
2. Filter for status="pending" entries
3. Group by trigger: "triage-suggestion" tasks first (labeled "From triage"), then "manual" tasks
4. Display each task with: ID, description, task_type (if set), client_name (if set), and a hint: -> /task run {id}
5. Show format:

   ## Pending Tasks

   **From triage:**
     1. [{id}] {description} ({task_type})
        {client_name if set}
        -> /task run {id}

   **Manual:**
     (none or list)

   Run /task run id to execute, or /task description for a new task.

6. If no pending tasks: "No pending tasks. Run /task description to create one, or /triage-inbox to scan for action items."

MODE 2: /task natural language description -- Parse, create, execute immediately (per D-09, TASK-01)

1. Parse the description to detect task_type using keyword patterns:
   - Contains "review contract" / "review agreement" / "check the contract" / "legal review" / "NDA" / "terms of service" -> task_type = "contract-review"
   - Contains "prep for meeting" / "prepare for" / "meeting with" / "agenda for" / "talking points" -> task_type = "meeting-prep"
   - Contains "summarize" / "summary of" / "key points" / "what does X say" / "review the doc" -> task_type = "document-summary"
   - Contains "draft reply" / "draft email" / "write a response" / "draft proposal" / "write to" -> task_type = "draft-comms"
   - If no pattern matches -> task_type = "document-summary" (safest default)

2. Resolve source email if description references a person or email:
   - Search recent triage files in data/triage/ (latest file): look for matching from_name or subject keywords
   - Use jq: jq -r 'select(.from_name | test("keyword"; "i")) | .message_id' data/triage/{latest}.jsonl
   - If match found: set source_email to the message_id, set client_name from triage record if available

3. Generate task ID: task-{YYYY-MM-DD}-{NNN} where NNN is zero-padded sequence. Count existing entries for today in data/tasks/active.jsonl using: jq -r '.id' data/tasks/active.jsonl 2>/dev/null | grep "task-$(date +%Y-%m-%d)" | wc -l then add 1 and zero-pad to 3 digits.

4. Get current timestamp: ISO 8601 with timezone offset

5. Create task record JSON with ALL schema fields:
   - id, ts, status="pending", description, trigger="manual", source_email (or null), outcome=null, completed_at=null, task_type, output_dir=null, source_triage=null, client_name (or null), draft_id=null

6. Append to data/tasks/active.jsonl

7. Log feed entry: type="task", summary="Created task: {description}", level="info", trigger="manual", details={"task_id": "{id}", "task_type": "{type}"}

8. Immediately dispatch to task-executor subagent using the Task tool:
   - Agent: task-executor
   - Instruction: the full task record as JSON, followed by: "Execute this task following your system prompt step by step. Return the formatted summary when complete."

9. Display the subagent's returned summary directly (per D-11, displayed inline)

10. After display, suggest: "Run /task to see your task queue, or /status for an overview."

MODE 3: /task run task-id -- Execute a pending task from the queue

1. Read data/tasks/active.jsonl
2. Find the task record matching the given ID (case-sensitive exact match)
3. If not found: "Task {id} not found. Run /task to see pending tasks."
4. If found but status is not "pending": "Task {id} is already {status}. Only pending tasks can be executed."
5. If found and pending: dispatch to task-executor subagent (same as Mode 2 step 8)
6. Display the subagent's returned summary

MODE 4: /task list -- Show all active tasks

1. Read data/tasks/active.jsonl
2. Show ALL entries (pending, in-progress, completed) in a table format with columns: ID, Status, Type, Description, Created
3. Show counts: "{N} total, {N} pending, {N} completed, {N} in-progress"

Argument detection logic (at the top of the command):
- If $ARGUMENTS is empty -> Mode 1 (queue display)
- If $ARGUMENTS starts with "run " -> Mode 3 (extract task-id from remainder)
- If $ARGUMENTS equals "list" -> Mode 4
- Otherwise -> Mode 2 (natural language task creation + execution)
