   ```bash

   ```

2. Read data/tasks/active.jsonl to get existing source_email values for deduplication.

3. Read the triage file and identify actionable records: those with non-empty action_items array AND action_type != "none".

4. For EACH actionable record with non-empty action_items:

   a. DEDUP CHECK: If the record's message_id matches any existing task's source_email in active.jsonl, skip it. Specifically, message_id 19d17c8f98f99484 already exists (task-2026-03-23-001) and MUST be skipped.

      - "contract" -> "contract-review"
      - "meeting" -> "meeting-prep"
      - "invoice" -> "document-summary"

   c. GENERATE task ID: task-{YYYY-MM-DD}-{NNN} where NNN is zero-padded. Count existing tasks for today and increment. The first new task will be task-2026-03-23-002 (since 001 already exists).

   d. CREATE task record JSON with ALL 13 schema fields:
      - id: generated
      - ts: current ISO 8601 timestamp with timezone offset
      - status: "pending"
      - description: first action_item from the array, enriched with sender context -- "{action_item} (from {from_name} @ {client_name})" if client_name available, otherwise "{action_item} (from {from_name})"
      - trigger: "triage-suggestion"
      - source_email: the triage record's message_id
      - task_type: mapped from action_type
      - output_dir: null
      - source_triage: "data/triage/2026-03-23T004602.jsonl"
      - client_name: from triage record if available, else null
      - outcome: null
      - completed_at: null
      - draft_id: null

   e. APPEND to data/tasks/active.jsonl (one JSON line per task)

5. After all records processed, validate:
   ```bash

   ```

6. Count results:
   ```bash
   jq -r 'select(.trigger=="triage-suggestion") | .id' data/tasks/active.jsonl | wc -l
   ```
