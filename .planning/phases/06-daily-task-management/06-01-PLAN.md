**1. Create schemas/todo-record.json** following the task-record.json pattern (JSON Schema draft-07).

Required fields: id, ts, status, text, priority, trigger.
Optional (nullable) fields: due_date, category, recurring, completed_at, linked_task_id.

Schema design per D-01 and D-05:
- `id`: string, pattern `^todo-\d{4}-\d{2}-\d{2}-\d{3}$` (like task-YYYY-MM-DD-NNN)
- `ts`: string, ISO 8601 with timezone offset
- `status`: enum ["active", "completed"] (simple -- not a project management tool)
- `text`: string, the to-do description
- `priority`: enum ["high", "normal", "low"] (per D-01)
- `trigger`: enum ["manual", "recurring"] (how it was created)
- `due_date`: string or null, ISO date format YYYY-MM-DD (per D-01)
- `category`: string or null, enum ["business", "personal", "marketing", null] (per D-01)
- `recurring`: string or null, enum ["daily", "weekly", "weekdays", null] (per D-01 recurring support)
- `completed_at`: string or null, ISO 8601 timestamp
- `linked_task_id`: string or null, task ID if to-do spawned a Claude task (per D-06)

Set `"additionalProperties": false` to match existing schema convention.

**2. Update schemas/feed-entry.json** -- add "todo" to the `type` enum array so feed entries can track to-do operations. The current enum is `["triage", "task", "draft", "system", "command", "briefing"]` -- append "todo".

**3. Create data/todos/.gitkeep** -- ensure the directory exists in git (follows data/tasks/ pattern).

**4. Create .claude/commands/todo.md** with YAML frontmatter and 5 modes per D-04.

The command file MUST have YAML frontmatter delimiters at the top with these fields:
- description: "Add, complete, list, and prioritize personal to-do items"
- allowed-tools: "Read, Write, Edit, Bash(jq *), Bash(date *), Bash(wc *), Bash(mkdir *), Bash(bash scripts/*), Bash(git *)"

Follow the exact same frontmatter format as .claude/commands/task.md.

Mode detection from $ARGUMENTS:
- **Empty** -> Mode 1 (show active to-dos for today)
- **"list" or starts with "list "** -> Mode 2 (filterable list: `/todo list`, `/todo list completed`, `/todo list #business`)
- **Starts with "done "** -> Mode 3 (complete by text match or number per D-03)
- **Starts with "prioritize " or "pri "** -> Mode 4 (reprioritize: `/todo pri 3 !high`)
- **Otherwise** -> Mode 5 (add new to-do with natural language parsing per D-02)

**MODE 1: Show Today's To-Dos (no arguments)**
1. Read data/todos/active.jsonl
2. Filter for status="active"
3. Sort by priority (high first, then normal, then low), then by due_date (soonest first, null last)
4. Display numbered list with priority badge, text, due date, category
5. Show counts and hint commands at bottom

**MODE 2: List To-Dos (list [filter])**
1. Read data/todos/active.jsonl
2. Parse optional filter from arguments after "list": "completed", "all", "#category", "!priority"
3. Apply filter (default: active only)
4. Display table format with columns: #, Status, Pri, To-Do, Due, Category
5. Show counts at bottom.

**MODE 3: Complete To-Do (done text-or-number per D-03)**
1. Read data/todos/active.jsonl
2. Parse argument after "done ":
   - If purely numeric: match by position in active list (sorted same as Mode 1)
   - Otherwise: fuzzy text match against active to-do text fields (case-insensitive substring match)
3. If match found:
   - Update the record: set status="completed", completed_at=now
   - Rewrite active.jsonl with the updated record (use jq to update in place -- read all, modify matching, write back)
   - Log to data/feed.jsonl: `{"ts": "{now}", "type": "todo", "summary": "Completed to-do: {text}", "level": "info", "trigger": "manual", "details": {"todo_id": "{id}", "text": "{text}"}}`
   - Display: "Done: {text}"
4. If no match: "No active to-do matching '{input}'. Run `/todo` to see your list."
5. If multiple matches: show the matches and ask to be more specific.
6. Trigger dashboard rebuild (same pattern as /task post-execution).

**MODE 4: Reprioritize (prioritize/pri number !priority)**
1. Parse: extract number (position in active list) and new priority (!high, !normal, !low)
2. Read data/todos/active.jsonl, find the Nth active to-do
3. Update priority field
4. Rewrite active.jsonl
5. Display: "Reprioritized: {text} -> {new priority}"
6. Trigger dashboard rebuild.

**MODE 5: Add New To-Do (natural language per D-02)**
1. Parse $ARGUMENTS for:
   - Priority: `!high`, `!normal`, `!low` (default: normal). Remove from text.
   - Category: `#business`, `#personal`, `#marketing`. Remove from text.
   - Due date: `by {date}`, `due {date}`, `before {date}`. Parse relative dates (Friday, tomorrow, next week). Remove from text.
   - Recurring: `every day`, `every weekday`, `every week`, `daily`, `weekly`. Remove from text.
   - Remaining text = to-do description
2. Generate ID: `todo-{YYYY-MM-DD}-{NNN}` (same pattern as tasks -- count existing today's todos + 1, zero-pad to 3)
3. Create record with all schema fields
4. Append to data/todos/active.jsonl
5. Log to data/feed.jsonl: `{"ts": "{now}", "type": "todo", "summary": "Added to-do: {text}", "level": "info", "trigger": "manual", "details": {"todo_id": "{id}", "priority": "{pri}", "category": "{cat}"}}`
6. Display: "Added: {text} {priority badge} {due date if set} {category if set}"
7. If linked_task pattern detected (text contains "ask claude to" or "have claude"): suggest "Want to create a /task for this?"
8. Trigger dashboard rebuild.
