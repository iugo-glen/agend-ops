## Not Applicable — Infrastructure Phase

Phase 1 (Foundation) has **no frontend deliverables**. This phase produces:

- Google OAuth 2.0 configuration (Google Cloud Console settings)
- Hardened MCP server installation and registration (CLI commands)
- NDJSON schema definitions (JSON files in schemas/)
- Directory structure (folders and placeholder files)
- Custom Claude Code commands (markdown files in .claude/commands/)
- .gitignore and CLAUDE.md configuration

None of these artifacts involve visual rendering, user-facing screens, HTML, CSS, JavaScript, or any component that would benefit from a design contract.

### Where UI Begins

The first phase with frontend deliverables is **Phase 4: Dashboard**, which delivers a GitHub Pages mobile-first dashboard. Phase 4 has a `UI hint: yes` marker in the roadmap and covers requirements VIS-01 through VIS-04.

The UI-SPEC for Phase 4 should define:
- Spacing scale for the dashboard layout
- Typography for mobile-first glanceable text
- Color contract for status indicators (email counts, priority buckets, activity states)
- Copywriting for empty states (no emails, no tasks, no activity)
- Component inventory (email count card, starred queue list, activity feed, pending tasks)

### Design System Pre-Decision

RESEARCH.md established that the dashboard will be served from `docs/` via GitHub Pages. No component library, build toolchain, or design system has been selected yet. That decision belongs to Phase 4's UI-SPEC research. The technology choices for the dashboard are intentionally deferred.
