  <how-to-verify>
Complete these steps in Google Cloud Console (per D-01, use Agend Systems Workspace account; per D-02, Glen has admin access):

1. Go to https://console.cloud.google.com/
2. Create a new project (or select existing) named "Agend Ops MCP"
3. Go to APIs and Services > OAuth consent screen
4. Set User Type to **Internal** (per D-03 -- this eliminates 7-day token expiry and bypasses verification requirements for Workspace accounts)
5. Fill in: App name = "Agend Ops MCP", Support email = your Workspace email
6. Save the consent screen
7. Go to APIs and Services > Library
8. Search for "Gmail API" and click Enable
9. Search for "Google Drive API" and click Enable
10. Go to APIs and Services > Credentials
11. Click Create Credentials > OAuth client ID
12. Application type: **Desktop app**
13. Name: "Claude Code MCP"
14. Click Create and copy the Client ID and Client Secret
15. Set environment variables (add to ~/.zshrc):
    ```bash
    export GOOGLE_OAUTH_CLIENT_ID="your-client-id.apps.googleusercontent.com"
    export GOOGLE_OAUTH_CLIENT_SECRET="your-client-secret"
    ```
16. Run `source ~/.zshrc` to load the new variables

Type "done" when OAuth is configured and environment variables are set.
  </how-to-verify>

    - Running `echo $GOOGLE_OAUTH_CLIENT_SECRET` outputs a non-empty string

  <action>
Execute these steps in order:

1. Clone the hardened MCP server (if not already cloned):
   ```bash
   git clone https://github.com/c0webster/hardened-google-workspace-mcp.git ~/hardened-google-workspace-mcp
   ```

2. Install dependencies using uv (Glen has uv 0.8.4):
   ```bash
   cd ~/hardened-google-workspace-mcp && uv sync
   ```

3. Verify the hardened fork has no dangerous tools by checking the codebase:
   - Confirm `send_gmail_message` does NOT exist as an available tool
   - Confirm `share_drive_file` does NOT exist as an available tool
   - Confirm `create_gmail_filter` does NOT exist as an available tool
   - Confirm `delete` operations are NOT available
   Record the verification results.

4. Register the MCP server with Claude Code at USER scope (not project scope -- per Pitfall 3):
   ```bash
   claude mcp add hardened-workspace \
     --scope user \
     -e GOOGLE_OAUTH_CLIENT_ID="$GOOGLE_OAUTH_CLIENT_ID" \
     -e GOOGLE_OAUTH_CLIENT_SECRET="$GOOGLE_OAUTH_CLIENT_SECRET" \
     -- uv run --directory ~/hardened-google-workspace-mcp python -m main --single-user
   ```

5. Check for conflicting Anthropic connectors (per Pitfall 5):
   ```bash
   claude mcp list
   ```
   If `claude.ai Gmail` appears, attempt to remove it:
   ```bash
   claude mcp remove "claude.ai Gmail"
   ```
   If removal fails or is not possible, document this in the summary.

6. Verify the MCP server connects:
   ```bash
   claude mcp list
   ```
   Expect to see: `hardened-workspace` in the list with status connected or needs-authentication.

7. If the server shows "Needs authentication", the OAuth flow will need to be completed on first tool use. This is expected behavior -- the server will prompt for browser-based OAuth consent on first invocation.
  </action>
