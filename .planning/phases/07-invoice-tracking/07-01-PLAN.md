Create `.claude/commands/invoice.md` following the exact pattern from `.claude/commands/todo.md`. The command manages invoice records for Glen.

**Frontmatter** (YAML frontmatter block with description and allowed-tools fields):
- description: "Create, list, mark-paid, sync, and track invoices"
- allowed-tools: "Read, Write, Edit, Bash(jq *), Bash(date *), Bash(wc *), Bash(mkdir *), Bash(bash scripts/*), Bash(git *), mcp__xero__list-invoices, mcp__xero__list-contacts"

Note: includes Xero MCP tools for sync mode. Argument: $ARGUMENTS

**Mode Detection** (from $ARGUMENTS):
- Empty -> Mode 1 (show outstanding invoices)
- "list" or starts with "list " -> Mode 2 (filterable list)
- "mark-paid " or "paid " -> Mode 3 (mark invoice as paid)
- "sync" -> Mode 4 (Xero sync)
- "overdue" -> Mode 5 (show overdue only)
- "link " -> Mode 6 (link a reminder to a Xero invoice)
- Otherwise -> Mode 7 (create new invoice with natural language)

**MODE 1: Show Outstanding Invoices (no arguments)**
1. Read `data/invoices/active.jsonl`
2. Filter for entries where status is NOT "paid", "written-off"
3. Compute overdue: any record where `status == "sent"` AND `due_date < today` (per D-02). Do NOT write overdue to the status field.
4. Sort: overdue first (by how many days overdue, descending), then sent, then draft, then reminder. Within each group, sort by due_date ascending (soonest first).
5. Display with project_code prominent (per D-08, user specifics):

```
## Outstanding Invoices

OVERDUE:
  1. [!] PROP-0324 | PROP Pty Ltd | $5,500.00 AUD | Due 2026-03-15 (8 days overdue) | INV-0042
  2. [!] ACME-0401 | Acme Corp | $2,200.00 AUD | Due 2026-03-20 (3 days overdue)

SENT (awaiting payment):
  3. CLUB-0312 | Sporting Club | $3,300.00 AUD | Due 2026-04-01 | INV-0043

DRAFT:
  4. NEW-0401 | New Client | $1,100.00 AUD | No due date

REMINDERS (need to invoice):
  5. [?] -- | Smith & Co | Work completed per email | No amount yet

{N} outstanding. Run /invoice paid N to mark paid, /invoice sync to refresh from Xero.
```

**MODE 2: List (list [filter])**
Filters: "all", "paid", "reminders", "sent", "draft", or client name substring.
Show table format with: ID, Status, Client, Project Code, Amount, Due Date, Invoice#.

**MODE 3: Mark Paid (mark-paid/paid text-or-number)**
1. Get outstanding invoices sorted same as Mode 1
2. Parse argument: numeric = position (1-indexed), else substring match on client_name or project_code
3. If single match: update status to "paid", set completed_at to now
4. Use jq to rewrite the record in active.jsonl (same pattern as todo done)
5. Log to data/feed.jsonl: type "invoice", summary "Marked invoice paid: {client} {project_code} ${amount}"
6. Trigger Post-Mutation

**MODE 4: Xero Sync (sync)**
1. Call `mcp__xero__list-invoices` to get all invoices with status AUTHORISED (outstanding)
2. Also call for status DRAFT and PAID (last 30 days for paid) if needed
3. For each Xero invoice:
   a. Map Xero status to local status using the mapping from research:
      - DRAFT/SUBMITTED -> "draft"
      - AUTHORISED -> "sent"
      - PAID -> "paid"
      - VOIDED -> "written-off"
      - DELETED -> skip/remove
   b. Check if `xero_invoice_id` already exists in local NDJSON
   c. If exists: update status, amount, amount_paid, due_date, invoice_number from Xero data
   d. If not exists: create new record with trigger "xero-sync", mapping:
      - Contact.Name -> client_name
      - Reference -> project_code (if matches pattern [A-Z]{2,5}-\d{2,5}, per research)
      - Total -> amount
      - TotalTax -> tax_amount
      - DueDate -> due_date
      - InvoiceNumber -> invoice_number
      - AmountPaid -> amount_paid
      - CurrencyCode -> currency
      - FullyPaidOnDate -> completed_at (if paid)
4. Report: "Synced {N} invoices from Xero. {N} new, {N} updated, {N} overdue."
5. Trigger Post-Mutation

**MODE 5: Overdue (overdue)**
1. Read active.jsonl, filter status=="sent" AND due_date < today
2. Sort by days overdue descending
3. Display with red emphasis and days-overdue count
4. Show total overdue amount

**MODE 6: Link (link reminder-number xero-invoice-id)**
1. Find the reminder record by position
2. Set xero_invoice_id on the record
3. Update status from "reminder" to match the Xero invoice status (call list-invoices filtered by ID if needed)
4. Log to feed
5. Trigger Post-Mutation

**MODE 7: Create New Invoice (natural language)**
Parse from $ARGUMENTS:
- Client name (first unrecognized segment)
- Amount: "$5500", "5500 AUD", "$2,200" patterns
- Currency: "AUD", "USD", "NZD" (default AUD per D-07)
- Due date: "by {date}", "due {date}", "net 30" (compute from today + 30)
- Project code: pattern matching [A-Z]{2,5}-\d{2,5} per D-08
- Description: remaining text after extraction

Generate ID: `inv-{YYYY-MM-DD}-{NNN}` (next available for today). Create record with ALL 18 schema fields, trigger "manual", status "draft". Append to active.jsonl. Log to feed. Trigger Post-Mutation.
