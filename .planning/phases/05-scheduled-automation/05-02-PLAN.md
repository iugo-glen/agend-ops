1. **Create directory structure:**
   ```bash
   mkdir -p .github/workflows
   ```

2. **Create `.github/workflows/daily-triage.yml`** with the following structure:

   **Name:** `Scheduled Triage`

   **Triggers:**
   ```yaml
   on:
     schedule:
       # All times ACST (UTC+9:30) mapped to UTC for weekdays Mon-Fri ACST
       # Per D-01: every 2 hours during business hours
       # Per D-02: ACST timezone (Adelaide)
       # Per D-03: weekdays only
       - cron: '30 21 * * 0-4'   # 7:00 ACST (Mon-Fri) = 21:30 UTC (Sun-Thu)
       - cron: '30 23 * * 0-4'   # 9:00 ACST (Mon-Fri) = 23:30 UTC (Sun-Thu)
       - cron: '30 1 * * 1-5'    # 11:00 ACST (Mon-Fri) = 01:30 UTC (Mon-Fri)
       - cron: '30 3 * * 1-5'    # 13:00 ACST (Mon-Fri) = 03:30 UTC (Mon-Fri)
       - cron: '30 5 * * 1-5'    # 15:00 ACST (Mon-Fri) = 05:30 UTC (Mon-Fri)
       - cron: '30 7 * * 1-5'    # 17:00 ACST (Mon-Fri) = 07:30 UTC (Mon-Fri)
     workflow_dispatch: {}         # Manual trigger for testing
   ```

   **Permissions:**
   ```yaml
   permissions:
     contents: write
   ```

   **Job: `triage`**
   - `runs-on: ubuntu-latest`
   - `timeout-minutes: 15`

   **Steps:**

   a. **Checkout:**
   ```yaml
   - uses: actions/checkout@v4
     with:
       token: ${{ secrets.PAT_TOKEN }}
   ```
   Use PAT_TOKEN (not default GITHUB_TOKEN) to ensure push permissions work on cron triggers (workaround for OIDC bug #814). The PAT must have Contents: Read & Write on this repo.

   b. **Configure git identity** (for commits from the runner):
   ```yaml
   - name: Configure git
     run: |
       git config user.name "agend-ops-bot"
       git config user.email "agend-ops-bot@users.noreply.github.com"
   ```

   c. **Install hardened MCP server:**
   ```yaml
   - name: Install hardened-workspace MCP
     run: |
       git clone --depth 1 https://github.com/c0webster/hardened-google-workspace-mcp.git /tmp/hardened-mcp
       cd /tmp/hardened-mcp
       pip install -e . || pip install uv && uv sync
   ```

   d. **Create MCP config file:**
   ```yaml
   - name: Create MCP config
     run: |
       cat > /tmp/mcp-config.json << 'EOF'
       {
         "mcpServers": {
           "hardened-workspace": {
             "command": "python",
             "args": ["-m", "hardened_google_workspace_mcp"],
             "cwd": "/tmp/hardened-mcp",
             "env": {
               "GOOGLE_OAUTH_CLIENT_ID": "${{ secrets.GOOGLE_CLIENT_ID }}",
               "GOOGLE_OAUTH_CLIENT_SECRET": "${{ secrets.GOOGLE_CLIENT_SECRET }}",
               "GOOGLE_REFRESH_TOKEN": "${{ secrets.GOOGLE_REFRESH_TOKEN }}"
             }
           }
         }
       }
       EOF
   ```
   NOTE: The exact command/args for the MCP server may need adjustment during testing. The research suggests `uv --directory ./hardened-mcp run main.py` as an alternative. The executor should check the hardened-mcp repo's README for the correct entry point and adjust accordingly.

   e. **Run Claude Code with triage + conditional briefing:**
   ```yaml
   - name: Run triage
     uses: anthropics/claude-code-action@v1
     with:
       anthropic_api_key: ${{ secrets.ANTHROPIC_API_KEY }}
       prompt: |
         You are running as a scheduled automation on a GitHub Actions runner.
         Environment: GITHUB_ACTIONS=true, trigger=scheduled.

         1. Run /triage-inbox to scan and categorize emails.

         2. Check if today's briefing already exists:
            ls data/briefings/$(date +%Y-%m-%d).md 2>/dev/null
            If it does NOT exist, this is the first triage run of the day.
            Run /daily-briefing to generate the morning briefing.

         3. After all operations complete, ensure all data changes are committed and pushed:
            git add data/ docs/
            git commit -m "data: scheduled triage $(date +%Y-%m-%dT%H%M)" || true
            git pull --rebase origin main || git pull --rebase origin master
            git push

         If any step fails, log an error to data/feed.jsonl:
         echo '{"ts":"'$(date -u +%Y-%m-%dT%H:%M:%S+00:00)'","type":"system","summary":"Scheduled triage failed: [error description]","level":"critical","trigger":"scheduled"}' >> data/feed.jsonl
         Then commit and push the error log entry.
       claude_args: |
         --mcp-config /tmp/mcp-config.json
         --model claude-sonnet-4-6
         --max-turns 25
   ```

   f. **Add comments throughout the YAML** explaining:
   - The ACST-to-UTC timezone mapping (reference the table in 05-RESEARCH.md)
   - Why PAT_TOKEN is used instead of GITHUB_TOKEN (bug #814)
   - The "first run of the day" briefing logic (D-07)
   - Error logging approach (D-09)

3. **Handle the git pull/push race condition** (Pitfall 4 from research):
   The prompt instructs Claude to `git pull --rebase` before pushing. This handles the case where Glen committed locally between scheduled runs. NDJSON append-only format means rebases almost never conflict.
