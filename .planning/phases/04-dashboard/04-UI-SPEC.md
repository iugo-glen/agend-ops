> Visual and interaction contract for the GitHub Pages dashboard. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

| Property | Value |
|----------|-------|

## Spacing Scale

Declared values (must be multiples of 4):

| Token | CSS Variable | Value | Usage |
|-------|-------------|-------|-------|
| xs | `--space-xs` | 4px | Icon gaps, badge padding, inline spacing |
| sm | `--space-sm` | 8px | Compact element spacing, card internal padding between lines |
| md | `--space-md` | 16px | Default element spacing, card padding, column gaps on mobile |
| lg | `--space-lg` | 24px | Section padding, card-to-card gap |
| xl | `--space-xl` | 32px | Column gaps on desktop, page horizontal padding |
| 2xl | `--space-2xl` | 48px | Header vertical padding |
| 3xl | `--space-3xl` | 64px | Not used in this phase — reserve for future pages |

Exceptions:
- Touch targets: minimum 44x44px tap area on all interactive elements (WCAG 2.5.5)
- Card border-radius: 8px (Notion-style rounded corners — D-05)
- Badge border-radius: 4px

### Accent Reserved For

Accent color (`--accent`) is exclusively used on these elements:
1. "Last updated" timestamp text (draws eye to freshness indicator — D-09)
2. Active column header underline on mobile tab navigation
3. Link text (if any hyperlinks appear in cards)
4. Count badges in column headers (e.g., "Starred (3)")

Accent is NOT used on card backgrounds, borders, or body text.

## Layout Contract

### Page Structure

```
+------------------------------------------+
|  [star] Agend Ops          Last updated:  |
|                             2 hours ago   |
+------------------------------------------+
|  Summary Bar                              |
|  [42 emails] [3 starred] [5 pending]     |
+------------------------------------------+
|  Starred | Urgent | Tasks | Activity      |  <-- mobile: horizontal scrollable tabs
+------------------------------------------+
|  Column content (cards stacked vertical)  |
+------------------------------------------+
```

### Mobile Layout (< 768px)

- **Single column, tabbed navigation** — tabs at top: Starred, Urgent, Tasks, Activity
- Active tab shows its card list; other columns hidden
- Tab bar: horizontal scroll if needed, sticky below header
- Cards: full width minus 16px horizontal padding each side
- Card stack: 8px gap between cards
- Header: sticky top, 48px height, page title left, "Last updated" right
- Summary bar: horizontal row of stat pills, 8px gap, scrollable if overflow

### Desktop Layout (>= 768px)

- **Four-column Kanban** — Starred | Urgent | Pending Tasks | Recent Activity
- Column min-width: 280px
- Column gap: 24px
- Page max-width: 1280px, centered
- Page horizontal padding: 32px
- Header: same as mobile but not sticky (more vertical space available)
- Summary bar: centered, stat pills in row

### Breakpoints

| Name | Value | Layout |
|------|-------|--------|
| Mobile | < 768px | Single column + tabs |
| Desktop | >= 768px | Four-column Kanban grid |

No intermediate breakpoint. Two layouts only for simplicity.

## Data Binding Contract

Dashboard fetches two JSON files at page load. No backend API.

### Data Sources

| File | URL | Source Schema | Refresh |
|------|-----|---------------|---------|
| `feed.json` | `./feed.json` (relative) | `schemas/feed-entry.json` | Page reload |
| `tasks.json` | `./tasks.json` (relative) | `schemas/task-record.json` | Page reload |

### Triage Data

The build script currently compiles feed.jsonl and active.jsonl. Triage data (email records for Starred/Urgent columns) is NOT yet compiled. The build script needs enhancement to also produce `docs/triage.json`.

**Required build script addition:**

| File | URL | Source | Content |
|------|-----|--------|---------|
| `triage.json` | `./triage.json` (relative) | Latest triage run file from `data/triage/` | Array of triage records from most recent run |

### Column-to-Data Mapping

| Column | Data Source | Filter | Sort |
|--------|------------|--------|------|
| Starred | `triage.json` | `starred === true` | `received` descending (newest first) |
| Urgent | `triage.json` | `priority === "urgent" && !starred` | `received` descending |
| Pending Tasks | `tasks.json` | `status === "pending" OR status === "in-progress"` | `ts` descending |
| Recent Activity | `feed.json` | All entries | `ts` descending (already sorted by build script) |

### Summary Bar Data

| Stat | Source | Computation |
|------|--------|------------|
| Emails scanned | `feed.json` | Latest triage entry's `details.emails_scanned` |
| Starred | `triage.json` | Count where `starred === true` |
| Pending tasks | `tasks.json` | Count where `status === "pending"` |
| Drafts created | `feed.json` | Latest triage entry's `details.drafts_created` |

### Last Updated

- Source: Latest entry in `feed.json` by `ts` field
- Display: Relative time ("2 hours ago", "just now", "yesterday")
- Computed in vanilla JS at page load (no library — simple relative time function)

## Interaction Contract

### Mobile Tab Switching

| Trigger | Action | Animation |
|---------|--------|-----------|
| Tap tab | Show corresponding column content, hide others | None — instant swap. No transitions. |
| Active tab | 2px accent bottom border, primary text color | Instant |

### Card Interactions

This is a **read-only dashboard**. No card tap actions, no modals, no drill-down, no delete, no edit. Cards are purely informational display.

| Interaction | Behavior |
|-------------|----------|
| Tap/click card | Nothing — no action (read-only per phase scope) |
| Hover card (desktop) | Subtle background shift to `var(--border)` at 50% opacity — 150ms transition |
| Long press (mobile) | Nothing — no context menu |
| Pull to refresh | Not implemented — use browser refresh |

### Page Load Sequence

1. Render HTML shell with header, summary bar (empty), tab bar, empty columns
2. Fetch `feed.json`, `tasks.json`, `triage.json` in parallel
3. On all fetched: populate summary bar stats, then populate each column's cards
4. Compute and display "Last updated: {relative time}"
5. If any fetch fails: show error state in affected columns only (other columns still render)

### Scroll Behavior

| Context | Behavior |
|---------|----------|
| Mobile column content | Native vertical scroll within visible tab |
| Desktop columns | Each column scrolls independently (overflow-y: auto, max-height: calc(100vh - 200px)) |
| Summary bar on mobile | Horizontal scroll (overflow-x: auto) |

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Color contrast | All text meets WCAG 2.1 AA (4.5:1 body, 3:1 large text) in both modes |
| Dark mode | Automatic via `prefers-color-scheme` media query — no manual toggle |
| Touch targets | All tappable elements (tabs) minimum 44x44px |
| Screen reader | Semantic HTML: `<header>`, `<nav>`, `<main>`, `<section>`, `<article>` |
| Tab navigation | Tabs use `role="tablist"`, `role="tab"`, `aria-selected`, `aria-controls` |
| Live region | Summary bar uses `aria-live="polite"` to announce data load |
| Reduced motion | Respect `prefers-reduced-motion: reduce` — disable hover transitions |
| Language | `<html lang="en">` |

## File Output Contract

All dashboard files live in `docs/` for GitHub Pages deployment.

| File | Purpose |
|------|---------|
| `docs/index.html` | Single-page dashboard (HTML + inline CSS + inline JS) |
| `docs/feed.json` | Activity feed data (compiled by build script) |
| `docs/tasks.json` | Task queue data (compiled by build script) |
| `docs/triage.json` | Triage email data (NEW — needs build script enhancement) |
