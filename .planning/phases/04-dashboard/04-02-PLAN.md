<!-- Data contracts for the three JSON files the dashboard fetches -->

docs/triage.json: Array of triage records (from schemas/triage-record.json)
  Required: message_id, thread_id, from, subject, received, priority
  Optional: from_name, starred, snippet, action_items, action_type, client_name, client_domain, draft_id
  Columns: starred===true -> Starred column, priority==="urgent" && !starred -> Urgent column

docs/tasks.json: Array of task records (from schemas/task-record.json)
  Required: id, ts, status, description, trigger
  Optional: task_type, output_dir, source_triage, client_name, outcome, completed_at, draft_id
  Filter: status==="pending" || status==="in-progress" -> Pending Tasks column

docs/feed.json: Array of feed entries (from schemas/feed-entry.json)
  Required: ts, type, summary, level, trigger
  Optional: details (object with action-specific metadata like emails_scanned, drafts_created), duration_ms
  All entries -> Recent Activity column, sorted by ts descending

Summary bar computations:
  - Emails scanned: latest triage-type entry in feed.json -> details.emails_scanned
  - Starred count: triage.json count where starred===true
  - Pending tasks: tasks.json count where status==="pending"
  - Drafts created: latest triage-type entry in feed.json -> details.drafts_created
  - Last updated: latest entry in feed.json by ts field
</interfaces>

  <action>
Create docs/index.html as a single self-contained file with ALL CSS and JS inline. No external stylesheets or script files (per D-06 and UI-SPEC file output contract).

The dashboard must implement EVERY specification from the UI-SPEC exactly. Here is the complete implementation specification:

**HTML structure:**
- `<!DOCTYPE html>`, `<html lang="en">` (accessibility contract)
- `<head>`: meta charset utf-8, meta viewport for mobile, `<meta name="color-scheme" content="light dark">` (prevents flash of wrong theme), `<title>Agend Ops - Dashboard</title>`, inline `<style>` block
- `<body>`: `<header>`, summary bar, tab bar (mobile), `<main>` with four columns, inline `<script>` block

**CSS (inline in `<style>`):**

1. CSS custom properties on `:root` -- use EXACT values from UI-SPEC Color section:
   - Light mode defaults: --surface: #FFFFFF, --surface-secondary: #F7F6F3, --text-primary: #37352F, --text-secondary: #787774, --border: #E3E2DE, --accent: #2383E2, --destructive: #EB5757
   - Spacing scale: --space-xs: 4px, --space-sm: 8px, --space-md: 16px, --space-lg: 24px, --space-xl: 32px, --space-2xl: 48px
   - Typography: --text-sm: 13px, --text-body: 15px, --text-heading: 18px, --text-display: 28px
   - Semantic colors for priorities and statuses (all values from UI-SPEC)
   - `color-scheme: light dark;`

2. Dark mode override via `@media (prefers-color-scheme: dark)` -- override ONLY the theme tokens (surface, text, border, accent) with dark values from UI-SPEC. NO JavaScript theme toggle (per D-04 and anti-patterns).

3. `@media (prefers-reduced-motion: reduce)` -- disable hover transitions (accessibility contract).

4. Layout:
   - Mobile (default): single column, header sticky top 48px, tab bar sticky below header, full-width cards with 16px horizontal padding, 8px card gap
   - Desktop (`@media (min-width: 768px)`): CSS Grid 4-column Kanban, `repeat(4, 1fr)` columns, 24px column gap, max-width 1280px centered, 32px horizontal page padding, hide tab bar, show all columns
   - Desktop columns: max-height `calc(100vh - 200px)`, overflow-y auto, `-webkit-overflow-scrolling: touch`

5. Components -- style each per UI-SPEC Component Inventory exactly:
   - Header: per spec (height, position, border, z-index, title left, last-updated right)
   - Summary bar: horizontal flex, gap 8px, overflow-x auto, pill shape per spec
   - Tab bar: flex row, mobile only, sticky, active tab styling per spec
   - Column: background surface-secondary, border-radius 8px, padding 16px, min-height 200px
   - Email card: per spec (background, border, border-radius, padding, line layout)
   - Task card: per spec (status dot 8px circle, description, type badge, timestamp)
   - Activity card: per spec (compact, type icon, summary, timestamp)
   - Priority badge: inline pill, white text, colored backgrounds per priority level
   - Action type badge: surface-secondary background, secondary text
   - Empty state: centered, secondary text color
   - Card hover on desktop: background shift to var(--border) at 50% opacity, 150ms transition

6. Subject line truncation: `-webkit-line-clamp: 2` for subject (2 lines), `1` for snippet (1 line). Use `-webkit-box` display with `overflow: hidden`.

**JavaScript (inline in `<script>`):**

1. Relative time function -- implement the EXACT format from UI-SPEC Copywriting Contract:
   - < 1 minute: "just now"
   - 1-59 minutes: "{N}m ago"
   - 1-23 hours: "{N}h ago"
   - 1-6 days: "{N}d ago"
   - >= 7 days: "Mar 17" format (abbreviated month + day)

2. Data fetching -- parallel fetch with graceful degradation:
   ```javascript
   const [feedResult, tasksResult, triageResult] = await Promise.allSettled([
     fetch('./feed.json').then(r => r.ok ? r.json() : Promise.reject(r.status)),
     fetch('./tasks.json').then(r => r.ok ? r.json() : Promise.reject(r.status)),
     fetch('./triage.json').then(r => r.ok ? r.json() : Promise.reject(r.status))
   ]);
   ```
   Extract values, defaulting to null on failure.

3. Summary bar population:
   - Emails scanned: find latest triage-type entry in feed array, read details.emails_scanned (default 0)
   - Starred: count triage records where starred === true
   - Pending tasks: count task records where status === "pending"
   - Drafts created: find latest triage-type entry in feed array, read details.drafts_created (default 0)
   - Use `aria-live="polite"` on summary bar container

4. Last updated: find the latest `ts` in feed.json, compute relative time, display in header right side in accent color. If no feed data, show "No data yet".

5. Column population:
   - **Starred column:** filter triage where `starred === true`, sort by `received` descending, render email cards
   - **Urgent column:** filter triage where `priority === "urgent" && !starred`, sort by `received` descending, render email cards
   - **Pending Tasks column:** filter tasks where `status === "pending" || status === "in-progress"`, sort by `ts` descending, render task cards
   - **Recent Activity column:** use all feed entries, already sorted by ts descending, render activity cards

6. Card rendering -- MUST use `document.createElement` and `element.textContent` for ALL data values. NEVER use innerHTML with data from JSON (XSS prevention per Research pitfall 2). Build each card type per the UI-SPEC Component Inventory:

   **Email card (createEmailCard function):**
   - Line 1: sender name (from_name || from) with star prefix (U+2605) if starred + priority badge right-aligned
   - Line 2: subject (truncated 2 lines via CSS)
   - Line 3: snippet (truncated 1 line via CSS, text-secondary)
   - Line 4: relative timestamp + action_type badge if action_type !== "none" and action_type is truthy

   **Task card (createTaskCard function):**
   - Line 1: status dot (8px colored circle per status) + description (truncated 2 lines)
   - Line 2: task type badge (use Task Type Labels from copywriting contract) + client_name if present
   - Line 3: created timestamp (relative time from ts field)

   **Activity card (createActivityCard function):**
   - Line 1: type icon (triage: U+1F4E5, task: U+2713, draft: U+270F, system: U+2699, command: U+2699) + summary text
   - Line 2: relative timestamp + duration if duration_ms present (format as "{N}s" or "{N}m {N}s")

7. Empty states -- when a column has zero items after filtering, render the empty state copy from the Copywriting Contract:
   - Starred: "No starred emails. Inbox zero!"
   - Urgent: "No urgent emails right now."
   - Tasks: "No pending tasks."
   - Activity: "No activity recorded yet. Run /triage-inbox to get started."

8. Error states -- if a fetch fails (result.status === 'rejected'), show in the affected columns:
   - "Couldn't load dashboard data. Try refreshing the page."

9. Mobile tab switching -- implement WAI-ARIA tab pattern:
   - role="tablist" on nav, role="tab" on buttons, role="tabpanel" on sections
   - aria-selected, aria-controls, tabindex management
   - Arrow Left/Right keyboard navigation per ARIA APG Tabs Pattern
   - Click/tap to switch tabs (instant, no animation per interaction contract)
   - Default active tab: Starred

10. Badge labels -- use EXACT copywriting contract labels:
    - Priority: urgent -> "Urgent", needs-response -> "Needs Response", informational -> "Info", low-priority -> "Low"
    - Task status: pending -> "Pending", in-progress -> "In Progress", completed -> "Done", cancelled -> "Cancelled"
    - Task type: contract-review -> "Contract Review", meeting-prep -> "Meeting Prep", document-summary -> "Doc Summary", draft-comms -> "Draft Comms"
    - Action type: contract -> "contract", invoice -> "invoice", meeting -> "meeting", deadline -> "deadline"

11. Handle missing optional fields with optional chaining and defaults:
    - `record.from_name || record.from` for sender display
    - `record.client_name || ''` (don't show "undefined")
    - `record.task_type` may be null -- show nothing if null
    - `record.snippet || ''` for email snippet
