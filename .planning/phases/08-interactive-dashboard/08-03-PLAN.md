  <action>
1. Create `schemas/action-queue-entry.json`:
   - JSON Schema draft-07, $id: "action-queue-entry"
   - Required fields: id, ts, action, target_id, status, requested_by
   - id: string, pattern `^action-.*` (ISO timestamp suffix)
   - ts: string (ISO 8601)
   - action: enum ["mark-paid", "complete-todo", "complete-task", "trigger-triage"]
   - target_id: string (the record ID to act on, empty for trigger-triage)
   - status: enum ["queued", "completed", "failed"]
   - requested_by: string (email address)
   - Optional: processed_at (string, ISO 8601), result (string, human-readable outcome)
   - additionalProperties: false (matching project convention)

2. Create `data/queue/.gitkeep` to ensure the directory exists in the repo.

3. Create `.claude/commands/process-queue.md`:
   - YAML frontmatter: description "Process pending actions from dashboard queue", allowed-tools (file read/write, bash for jq)
   - Step-by-step instructions for Claude:

   **Step 1: Read queue**
   Read `data/queue/actions.jsonl`. If file doesn't exist or is empty, report "No pending actions" and exit.

   **Step 2: Filter pending**
   Parse each line. Filter to entries where `status === "queued"`. If none, report "No pending actions" and exit.

   **Step 3: Process each action**
   For each queued action, execute based on action type:

   - `mark-paid`: Read `data/invoices/active.jsonl`, find the invoice matching `target_id`. Update its status to "paid", set completed_at to current ISO timestamp. Rewrite the file. Log to feed.jsonl: type "command", summary "Marked invoice {id} as paid (dashboard action)", trigger "hook".

   - `complete-todo`: Read `data/todos/active.jsonl`, find the todo matching `target_id`. Update status to "completed", set completed_at to current ISO timestamp. Rewrite the file. Log to feed.jsonl: type "todo", summary "Completed to-do: {text} (dashboard action)", trigger "hook".

   - `complete-task`: Read `data/tasks/active.jsonl`, find the task matching `target_id`. Update status to "completed", set completed_at to current ISO timestamp. Rewrite the file. Log to feed.jsonl: type "task", summary "Completed task: {description} (dashboard action)", trigger "hook".

   - `trigger-triage`: Run the /triage-inbox flow. Log to feed.jsonl: type "triage", summary "Triage triggered from dashboard", trigger "hook".

   **Step 4: Update queue entries**
   For each processed action: update status to "completed", set processed_at to current timestamp, set result to a description of what was done. Move completed entries from `data/queue/actions.jsonl` to `data/queue/processed.jsonl` (append). Remove them from actions.jsonl. If action failed, set status to "failed" with error in result, leave in actions.jsonl.

   **Step 5: Rebuild dashboard data**
   Run `scripts/build-dashboard-data.sh` to update dashboard JSON files.

   **Step 6: Commit and report**
   Git add and commit all changed data files with message "feat: process {N} dashboard actions". Report summary: how many actions processed, what was done.
  </action>
